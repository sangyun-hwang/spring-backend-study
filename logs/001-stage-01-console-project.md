# 001. Stage 01 Java 콘솔 프로젝트

날짜: 2026-05-08

단계: Stage 01 Java 콘솔 프로젝트

## 오늘 한 것

- `StudyLog` 클래스로 공부 기록 하나를 표현했다.
- `StudyCategory` enum으로 카테고리를 문자열 대신 안전하게 표현했다.
- `StudyLogManager`를 만들어 공부 기록 목록 관리 로직을 분리했다.
- `List<StudyLog>`와 `ArrayList`로 여러 공부 기록을 다뤘다.
- 전체 기록 조회, 카테고리별 기록 조회, 카테고리별 공부 시간 합계를 구현했다.
- `Scanner`로 콘솔 입력을 받아 공부 기록을 추가하는 메뉴 프로그램을 만들었다.
- 잘못된 title과 minutes가 들어오지 않도록 `StudyLog` 생성자에서 입력값을 검증했다.

## 새로 이해한 것

- 클래스는 객체를 만들기 위한 설계도이고, 객체는 실제 데이터를 가진 인스턴스다.
- `private final` 필드는 외부에서 직접 바꿀 수 없고, 생성 후 재할당도 막는다.
- 생성자에서는 필드에 값을 넣기 전에 먼저 유효한 값인지 검증하는 것이 자연스럽다.
- 배열은 크기가 고정되어 있고, `ArrayList`는 값을 추가하면서 목록을 늘릴 수 있다.
- `List<StudyLog>`는 `StudyLog` 객체만 담을 수 있는 목록이라는 뜻이다.
- `enum`은 정해진 선택지 중 하나를 표현할 때 문자열보다 안전하다.
- `StudyLogManager`로 로직을 분리하면 `Main`은 실행 흐름과 출력에 더 집중할 수 있다.
- `nextInt()` 다음에는 입력 버퍼에 남은 Enter를 처리하기 위해 `nextLine()`을 한 번 더 호출해야 한다.

## 아직 헷갈리는 것

- `Main`이 어디까지 맡고, `StudyLogManager`가 어디까지 맡아야 하는지의 경계가 아직 완전히 익숙하지 않다.
- `Scanner` 입력 예외 상황을 더 부드럽게 처리하는 방법은 아직 익숙하지 않다.
- 반환하는 메서드와 직접 출력하는 메서드를 어떤 기준으로 나눌지 더 연습이 필요하다.

## 직접 설명해보기

이번 프로젝트는 공부 기록을 콘솔에서 추가하고 조회하는 작은 프로그램이다.

`StudyLog`는 공부 기록 하나를 표현한다. 제목, 카테고리, 공부 시간, 메모를 필드로 가지고, 생성자에서 잘못된 제목이나 공부 시간을 막는다.

`StudyCategory`는 카테고리를 정해진 값으로 제한한다. 문자열을 직접 쓰면 오타가 나도 컴파일 전에 잡기 어렵지만, enum을 쓰면 `JAVA`, `GIT`, `SPRING`, `DATABASE`, `ETC` 중 하나만 사용할 수 있다.

`StudyLogManager`는 여러 공부 기록을 `List<StudyLog>`로 관리한다. 기록 추가, 전체 목록 반환, 카테고리별 목록 반환, 카테고리별 시간 합계를 담당한다.

`Main`은 사용자 입력을 받고 메뉴를 실행한다. 사용자가 선택한 메뉴에 따라 `StudyLogManager`에 일을 시키고, 반환된 결과를 콘솔에 출력한다.

## 막힌 지점

- `StudyLog.java`와 `Main.java`의 역할을 처음에 구분하지 못해서 클래스 밖에 실행 코드를 작성했다.
- `nextInt()` 다음에 `nextLine()`이 바로 실행되어 title이나 memo 입력이 건너뛰어지는 문제가 있었다.
- IntelliJ 실행 설정, Source Root, 파일 인코딩 문제 때문에 초반에 실행 버튼과 한글 출력에서 헷갈렸다.
- `List` 자체를 출력하면 원하는 `printInfo()` 형태가 나오지 않아서, for-each로 하나씩 출력해야 한다는 점을 배웠다.
