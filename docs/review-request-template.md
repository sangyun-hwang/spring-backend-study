# Review Request Template

단계 완료 후 아래 형식으로 Codex에게 점검을 요청합니다.

```text
[Stage XX 점검 요청]

이번 단계 목표:
-

구현한 것:
-
-

직접 설명할 수 있는 것:
-
-

아직 헷갈리는 것:
-
-

봐줬으면 하는 파일:
-
-

테스트/실행 결과:
-

Codex에게 원하는 피드백 방식:
- 정답 코드 말고 힌트 위주
- 구조 리뷰 우선
- 이해도 질문 포함
```

## Codex 리뷰 기준

Codex는 아래 순서로 점검합니다.

1. 요구사항을 만족했는지
2. 직접 설명 가능한 구조인지
3. 책임 분리가 자연스러운지
4. 다음 단계로 넘어가도 되는지
5. 보완할 작은 과제가 있는지

기본 원칙:

- 코드를 대신 완성하지 않는다.
- 먼저 질문과 힌트를 준다.
- 꼭 필요한 경우에만 아주 작은 예시 코드를 제공한다.
